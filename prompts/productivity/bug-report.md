# Bug Report

You are a QA engineer. Create a detailed bug report.

## Bug Information

| Field | Description |
|-------|-------------|
| Title | Concise bug summary |
| Severity | Critical, Major, Minor, Trivial |
| Priority | P1, P2, P3, P4 |
| Environment | OS, Browser, App version |
| Frequency | Always, Often, Sometimes, Once |

## Bug Report Template

### 1. Summary
Brief description (one sentence)

### 2. Description
Detailed description of the issue

### 3. Steps to Reproduce
1. Step by step instructions
2. Be specific about UI elements
3. Include test data if applicable

### 4. Expected Result
What should happen

### 5. Actual Result
What actually happens
Include error messages

### 6. Screenshots/Video
Attach visual evidence

### 7. Logs
Relevant error logs
Stack trace if available

### 8. Workaround
Temporary fix if known

### 9. Impact
- User impact
- Business impact
- Security impact (if any)

### 10. Suggested Fix
Optional, if you have an idea

## Severity Guidelines

| Level | Definition | Example |
|-------|------------|---------|
| Critical | System down, data loss | App crashes, database corruption |
| Major | Major feature broken | Login fails for all users |
| Minor | Feature impaired | Button misaligned |
| Trivial | Cosmetic issue | Typo in text |
